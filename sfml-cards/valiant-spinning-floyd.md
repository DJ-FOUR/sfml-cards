# 斗地主肉鸽卡牌游戏 — 策划报告

## 一、游戏概述

**游戏类型**：斗地主规则驱动的无尽闯关肉鸽卡牌游戏

**核心循环**：
```
选择角色 → 开局抽牌 → 对战敌人 → 清空手牌过关
    → 从随机3个技能中选1个获得 → 装备技能（最多3个）
    → 下一关（敌人继承你上一关的3个技能）→ 循环
```

**独特机制**：镜像敌人——你用什么技能打赢，下一关敌人就用什么技能对付你。越到后面，你在和"上一关的自己"对战。

---

## 二、运行结构

- **模式**：无尽模式（Endless Run）
- **关卡推进**：线性，一关接一关，无分支
- **每关**：玩家和敌人各随机发20张牌（54张标准扑克牌）
- **过关条件**：玩家清空手牌
- **失败条件**：敌人先清空手牌
- **死亡**：Run结束，从第1关重新开始
- **局外成长**：无（纯Roguelike）

---

## 三、角色系统（3个）

### 角色1：争锋者
- **固定被动技 "强袭"**：每回合首次出牌额外获得1点能量
- **风格**：进攻型，能量获取快，适合频繁释放技能

### 角色2：谋略家
- **固定被动技 "谋定"**：每关首次使用技能不触发冷却
- **风格**：灵活型，战斗中可变通调整策略

### 角色3：掌控者
- **固定被动技 "储备"**：手牌上限+2（初始手牌22张而非20张）
- **风格**：控场型，牌多选择多，持久战优势

---

## 四、技能系统

### 4.1 技能规则

- **获取**：击败敌人后，从随机3个技能中选择1个获得
- **装备**：每关开始前，从已获得的技能中选择最多3个装备到技能槽
- **敌人技能**：下一关敌人自动装备你上一关使用的3个技能（镜像机制）
- **第1关**：敌人无技能
- **资源**：初始3点能量，每回合+1能量（争锋者被动额外+1），能量上限10
- **释放**：在己方回合任意时机释放主动技能（出牌前/后）
- **冷却**：有冷却的技能释放后进入冷却，冷却在每回合开始时-1

## 五、敌人系统

### 5.1 基础AI
- 斗地主标准AI：跟牌时出最小的合法牌，自由出牌时从最小单牌开始
- 随着关卡提升，AI基础智能略微增强（更倾向于先出大牌型清牌）

### 5.2 镜像技能机制
- 第1关：敌人无技能
- 第2关起：敌人自动装备玩家上一关使用的3个技能
- 如果玩家上一关只用了1-2个技能，敌人只有对应数量的技能
- 敌人使用技能不消耗能量（纯CD制），冷却与玩家一致
- 敌人优先使用干扰类技能，其次资源类，最后增强类

### 5.3 难度曲线
- 关卡1-5：敌人基础AI，镜像技能简单
- 关卡6-10：敌人手牌+1（21张），AI更激进
- 关卡11-15：敌人手牌+2（22张），镜像技能冷却-1
- 关卡16+：敌人手牌+3（23张），每回合自动回复能量（用于技能释放）
- 敌人手牌最多24张封顶

## 六、各界面及交互

### 6.1 主菜单
- 标题："影子卡牌"
- 按钮："开始游戏"、"选择角色"、"退出"

### 6.2 角色选择界面
- 3个角色卡片展示（头像、名称、固定被动技描述）
- 点击角色 → 进入Run

### 6.3 关卡过渡界面（每关前）
- 显示：第N关
- 已获得的技能列表（高亮已装备的3个）
- 技能槽×3，点击装备/替换
- 显示下一关预览：敌人将继承的技能（即当前已装备的技能）
- 按钮："开始战斗"

### 6.4 奖励界面（过关后）
- 随机3个技能卡片展示
- 点击选择1个 → 进入下一关过渡界面
- 如果已拥有该技能 → 显示"已拥有"标记，可重选

### 6.5 战斗界面（改造现有Game界面）
- 顶部：敌人信息（名称、手牌数、已装备技能图标）
- 中部：弃牌/出牌区域
- 下方：玩家手牌（可点击选择）
- 底部栏：技能槽×3（显示技能图标+冷却状态+能量消耗），能量条，出牌/不出按钮

### 6.6 Run结束界面
- 显示：到达第N关，获得技能总数
- 按钮："再来一局"、"返回主菜单"

## 七、技术实现要点

### 7.1 需新增的模块
1. **`skill.hpp/cpp`** — 技能数据结构、技能池定义、技能效果接口
2. **`character.hpp/cpp`** — 角色数据结构、3个角色定义
3. **`enemy.hpp/cpp`** — 敌人AI增强、镜像技能逻辑、难度递进
4. **`run_state.hpp/cpp`** — Run全局状态管理（当前关卡、已获得技能、已装备技能等）

### 7.2 需改造的模块
1. **`game_state.hpp/cpp`** — 融入技能系统调用、能量管理、冷却管理
2. **`renderer.hpp/cpp`** — 新增角色选择、技能选择、奖励选择、战斗技能UI
3. **`main.cpp`** — 新增大屏状态机（角色选择、过渡、奖励、Run结束界面）
4. **`save.hpp/cpp`** — 可能废弃（纯肉鸽无存档需求），或改为记录最高关卡

### 7.3 需新增的资源
- 技能图标图片（或使用文字/符号替代）
- 角色头像图片（或使用文字/形状替代）

## 八、验证方式

1. **编译通过** — CMake构建，C++17 + SFML 3.0.2
2. **Run流程** — 从主菜单 → 选角色 → 第1关战斗 → 过关选技能 → 装备技能 → 第2关（确认敌人继承了技能）→ 重复至失败
3. **技能效果验证** — 逐一测试每个技能在使用后产生预期效果
4. **镜像验证** — 确认第N关的敌人技能 = 玩家第N-1关装备的3个技能
5. **边界测试** — 技能冷却倒计时、能量不足时技能不可用、手牌为0时过关判定
