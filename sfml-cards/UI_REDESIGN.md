# 斗牌Rogue UI 视觉重设计方案

> **风格定位**：赛博朋克军事终端 / 战术OS界面  
> **色彩主轴**：荧光绿 `#CCFF00` + 纯白 `#FFFFFF` + 纯黑 `#000000`  
> **设计原则**：仅替换视觉表现，不改动游戏逻辑、交互逻辑、碰撞检测区域与代码结构。

---

## 一、全局色彩与材质系统

### 1.1 主色板（替换 `renderer.cpp` 匿名命名空间常量）

| 语义 | 原色值 | 新色值 | 用途 |
|------|--------|--------|------|
| 背景底色 | `#2D2D2D`（图片） | `#050505` 纯黑 | 清除色、遮罩底色 |
| 主荧光绿 | — | `#CCFF00` | 边框高亮、选中态、能量、标题强调 |
| 暗绿背景 | — | `#1A2A0A` | 按钮填充、面板底色 |
| 深灰面板 | `#404050` | `#111111` | 卡片背景、技能槽底色 |
| 边框常规 | `#646464` | `#333333` | 未激活边框 |
| 边框高亮 | `#C8C864` | `#CCFF00` | 悬停边框、发光 |
| 文字主色 | `#FFFFFF` | `#FFFFFF` | 标题、技能名 |
| 文字次要 | `#C8C8C8` | `#AAAAAA` | 描述、冷却信息 |
| 文字禁用 | `#808080` | `#555555` | 不可用状态 |
| 能量填充 | `#FFC800` | `#CCFF00` | 能量条填充 |
| 能量背景 | `#3C3C3C` | `#222222` | 能量条槽 |
| 警示红 | `#FF5050` | `#FF3333` | 敌人技能、失败文字 |
| 选中overlay | `#FFFF64` (alpha100) | `#CCFF00` (alpha80) | 手牌选中高亮 |

### 1.2 新增材质规范

- **警示斜条纹**：宽 8px，角度 45°，颜色 `#CCFF00` + `#000000` 交替。用于关键按钮（出牌、开始战斗）的顶部或底部装饰条。
- **角标切角**：所有主要面板（技能卡片、角色卡片、按钮）的四个角做 8px 的 45° 切角，形成八边形轮廓，而非纯矩形。
- **扫描线纹理**：在纯黑背景上叠加 1px 间隔的 `#0A0A0A` 水平线（alpha 30%），模拟 CRT 显示器效果。使用 `sf::RectangleShape` 循环绘制或预渲染为 `sf::RenderTexture`。
- **终端边框**：重要信息框使用 2px 外边框 + 1px 内边框（内缩 4px），外框 `#CCFF00`，内框 `#333333`。

## 二、字体与文字规范

### 2.1 字体替换策略

当前使用 `simsun.ttc`（宋体）。新风格需匹配粗黑/无衬线力量感：

| 层级 | 建议字体（Windows预装） | 字号比例 | 样式 |
|------|------------------------|----------|------|
| 大标题 | `Microsoft YaHei` 或 `SimHei` | `h * 0.08` | **Bold**，可倾斜 `sf::Text::Italic` |
| 面板标题 | 同上 | `h * 0.04` | Bold，增加英文后缀 `// TITLE` |
| 按钮文字 | 同上 | `h * 0.03` | Bold |
| 技能名/角色名 | 同上 | 相对原尺寸 +20% | Bold，白色 |
| 描述/数据 | 同上 | 不变 | Regular，`#AAAAAA` |
| 角标数字 | `Arial` 或等宽字体 | 相对 | Bold，荧光绿 |

### 2.2 双语标注规范（新增）

为强化军事OS感，所有主要区域标题采用 **"中文 // ENGLISH"** 格式：

- 主菜单：`斗牌ROGUE // CARD ROGUE`
- 战场状态栏：`战场 // BATTLEFIELD`
- 你的回合：`你的回合 // YOUR TURN`
- 敌人回合：`敌方运算中 // ENEMY PROCESSING`
- 能量：`能量 // ENERGY`
- 被动技能：`被动协议 // PASSIVE`
- 技能槽：`装备槽 // SLOTS`
- 出牌按钮：`执行 // EXECUTE`
- 不出按钮：`跳过 // SKIP`
- 返回按钮：`返回 // BACK`
- 战斗：`开始战斗 // ENGAGE`
- 失败：`任务失败 // MISSION FAILED`
- 关卡：`第 N 关 // LEVEL N`

> 实现方式：将原有 `sf::Text` 的字符串直接替换为双语格式，或使用两个 `sf::Text` 分层绘制（中文主标题偏左/上，英文副标题偏右/下，字号 40%）。

## 三、分界面改造方案

### 3.1 全局背景

**现状**：拉伸 `back0.png`（一张静态图）。

**新方案**：
1. 弃用 `back0.png`，改为代码绘制纯黑背景 `#050505`。
2. 叠加低透明度网格线：
   - 垂直线：每 80px 一条 `#1A1A1A`（alpha 40%）
   - 水平线：每 80px 一条 `#1A1A1A`（alpha 40%）
   - 形成战术地图网格感。
3. 四角添加 L 形荧光绿角标（长度 40px，线宽 2px），标记安全区域。
4. 底部右侧增加固定文字 `v1.0.0_OS`（`#333333`，小号）。

> **注意**：`drawBackground()` 保留原有 `m_bgTexture` 加载逻辑以防兼容，但绘制时优先使用新代码背景；或在 `initialize()` 中若 `back0.png` 加载失败则自动回退到代码背景。

---

### 3.2 主菜单（MainMenu）

**布局保持**：标题居中 + 3个垂直按钮（开始游戏 / 选择角色 / 退出）。

**视觉改造**：

1. **标题区**：
   - 原 `斗牌Rogue` → `斗牌ROGUE // TACTICAL CARD SYSTEM`
   - 字体大小放大至 `h * 0.10`
   - 增加荧光绿底部下划线：宽度 = 文字宽度的 1.2 倍，高度 3px，位于文字下方 8px。
   - 标题文字颜色 `#FFFFFF`，可添加微弱外发光（`sf::Text` 无法直接发光，可通过在下层绘制同色放大 102% 的副本并设 alpha 30% 模拟）。

2. **菜单按钮**：
   - 形状：由纯矩形改为 **切角矩形**（八边形）。切角 6px。
   - 尺寸：`bw = w * 0.40`，`bh = h * 0.07`（比原来更宽更扁，更有科技感）。
   - 颜色：
     - 常态：`#0A0A0A` 填充，`#333333` 2px 外边框。
     - 悬停：`#1A2A0A` 填充，`#CCFF00` 2px 外边框，同时按钮整体向右偏移 4px（位移动效）。
     - 禁用：`#0A0A0A` 填充，`#222222` 边框。
   - 文字：白色 Bold，增加 2px 的 `#000000` 描边（通过下层偏移 1px 的深色文字模拟）。
   - 左侧增加小型荧光绿方块指示器（6x6px），悬停时闪烁。

3. **版本信息**：
   - 右下角增加 `BUILD_2026.05 // DEFIANCE_OS v1.0`，`#444444`，`h * 0.018`。

---

### 3.3 角色选择（CharacterSelect）

**布局保持**：3张角色卡片横向排列 + 左上角返回按钮。

**视觉改造**：

1. **页面标题**：
   - 原 `选择角色` → `选择单位 // SELECT OPERATOR`
   - 增加荧光绿底线。

2. **角色卡片（`renderCharCardToRT`）**：
   - **形状**：切角矩形（切角 12px），尺寸保持 `280x480` 比例。
   - **底色**：常态 `#0D0D0D`，悬停 `#141E0A`。
   - **边框**：常态 2px `#333333`，悬停 2px `#CCFF00` + 外发光（在 RT 外层绘制一层 alpha 递减的多个边框模拟）。
   - **头像区**：原灰色方块 `#505078` → 改为 `#1A1A1A` 填充，边框 `#333333`。内部数字改为荧光绿 `#CCFF00`。未来替换为角色立绘后，立绘使用高对比度黑白滤镜 + 绿色色调叠加。
   - **角色名**：放大至 `ch * 0.075`，白色 Bold，增加英文代号如 `// VANGUARD`。
   - **被动技名**：`#CCFF00`，`ch * 0.055`，增加前缀标签 `[被动]` 使用方括号。
   - **被动描述**：`#AAAAAA`，增加行高。
   - **顶部角标**：卡片左上角增加荧光绿色块，内写 `OP-0{N}`（如 OP-01, OP-02, OP-03），`#000000` 文字。

3. **3D悬停倾斜**：保留原有透视倾斜逻辑，但悬停时的发光色从橙黄 `#F0A500` 改为荧光绿 `#CCFF00`。

4. **返回按钮**：与主菜单按钮风格统一，文字改为 `返回 // BACK`。

### 3.4 关卡过渡界面（Transition）

**布局保持**：左侧已获得技能列表 + 右侧装备槽 + 敌人预览 + 底部开始战斗按钮。

**视觉改造**：

1. **标题**：
   - 原 `第 N 关` → `第 N 关 // LEVEL N`
   - 增加荧光绿底线，标题区域增加警示斜条纹背景（高度 4px，位于标题上方）。

2. **左侧面板（已获得技能）**：
   - 列表项改为**细线分隔**风格，弃用独立矩形底色。
   - 每条技能：左侧增加荧光绿竖线指示器（2px 宽，高 = itemH * 0.6）。已装备的技能指示器常亮，未装备但悬停时闪烁。
   - 文字格式：`[技能名] // S0{N}` + 描述。能量与冷却用括号括起 `(费2 CD2)`，颜色 `#888888`。
   - 悬停态：整行背景变为 `#111A08`，左侧竖线变粗至 4px。

3. **右侧装备槽**：
   - 槽位形状：切角矩形，比例保持 7:12。
   - 空槽：内部显示 `-- EMPTY --`，颜色 `#333333`，中央增加小型十字准星图案（由两条 1px 线交叉绘制）。
   - 有技能：顶部增加荧光绿条（高度 8px）作为技能类型标识。技能名白色 Bold，冷却/能量信息在底部用等宽字体显示。
   - 悬停：边框 `#CCFF00`，轻微上浮 2px。

4. **敌人预览区**：
   - 标签改为 `敌方继承协议 // ENEMY INHERITANCE`，颜色 `#FF3333`（红色警示）。
   - 敌人技能用红色边框小卡片表示，与玩家绿色系形成对比。

5. **开始战斗按钮**：
   - 改为**满宽警示按钮**：宽度 `w * 0.55`，高度 `h * 0.075`。
   - 顶部和底部各增加 4px 高的黄黑斜条纹带。
   - 常态填充 `#1A2A0A`，边框 `#CCFF00` 2px。
   - 悬停：填充变为 `#CCFF00`，文字反黑 `#000000`，整体产生轻微脉缩放（1.0 → 1.03 循环）。
   - 文字：`开始战斗 // ENGAGE`。

### 3.5 奖励界面（Reward）

**布局保持**：3张技能卡片横向排列，居中。

**视觉改造**：

1. **标题**：
   - 原 `选择初始技能 / 选择奖励` → `获取协议 // ACQUIRE PROTOCOL` / `奖励结算 // REWARD`
   - 增加荧光绿底线。

2. **技能卡片（`drawSkillCard`）**：
   - **形状**：切角矩形（切角 10px），比例 7:12。
   - **底色**：常态 `#0D0D0D`，已拥有 `#080808`（更暗）。
   - **边框**：常态 2px `#333333`，悬停 2px `#CCFF00`。
   - **顶部标识条**：卡片顶部 12px 高度条带：
     - 常态：`#1A1A1A`
     - 悬停：`#CCFF00`
   - **图标区**：原灰色方块 → 改为线框风格的八角形边框 `#333333`，内部技能编号 `S0{N}` 荧光绿。悬停时边框变绿。
   - **技能名**：放大，白色 Bold，居中。
   - **描述**：缩小行间距，颜色 `#AAAAAA`。
   - **能量/冷却**：底部增加独立黑条区域（高度 22px），内部左对齐 `ENERGY: N`，右对齐 `CD: N`，颜色 `#888888`，等宽字体。
   - **已拥有标签**：原 `[已拥有]` → 改为顶部角标：荧光绿色块覆盖在卡片左上角，文字 `OWNED`，黑色粗体，倾斜。

3. **悬停发光**：保留现有发光逻辑，颜色从橙黄改为荧光绿 `#CCFF00`，alpha 计算方式不变。

### 3.6 游戏失败界面（GameOver）

**布局保持**：遮罩 + 大标题 + 统计 + 回到主菜单按钮。

**视觉改造**：

1. **遮罩**：`#000000` alpha 180（比原来更深）。

2. **标题**：
   - 原 `你输了` → `任务失败 // MISSION FAILED`
   - 颜色：由红 `#FF5050` 改为荧光绿反色警示：`#CCFF00` 文字 + 黑色粗描边，模拟系统报错。
   - 标题上方增加红色闪烁的 `ERROR` 字样（可用定时器切换 alpha 实现闪烁，无需新资源）。

3. **统计信息**：
   - 格式改为终端日志风格：
     ```
     > 到达关卡 : LEVEL {N}
     > 获得技能 : {N} PROTOCOLS
     > 存活时间 : --:--
     ```
   - 颜色 `#AAAAAA`，等宽排版，左侧对齐。
   - 每行左侧增加 `>` 荧光绿前缀。

4. **按钮**：`回到主菜单 // RETURN TO BASE`，使用主菜单按钮风格。

### 3.7 战斗界面（Game）—— 核心改造区

**布局保持**：
- 顶部：敌人手牌（牌背）+ 敌人技能槽
- 中上部：电脑出牌区
- 中部：玩家出牌区
- 中下部：出牌/不出按钮
- 底部：玩家手牌（扇形）
- 左下角：玩家技能槽
- 底部中央偏右：能量条
- 底部中央：状态文字
- 左上角：返回按钮 + 调试按钮

#### 3.7.1 顶部区域（敌人信息）

1. **敌人标签**：
   - 原 `镜像AI N 张` → `敌方单位 // ENEMY_AI`
   - 右侧增加手牌数量角标：荧光绿方块 `#CCFF00`，黑色数字，紧贴标签右侧。

2. **敌人技能槽**：
   - 位置保持顶部居中。
   - 底色：有技能时 `#1A0A0A`（暗红底），边框 `#FF3333` 1px。
   - 冷却中：整个槽位覆盖半透明 `#000000` alpha 120，中央红色 `CD{N}`。
   - 空槽：`#111111` 底，`#333333` 边框，中央红色 `--`。

#### 3.7.2 出牌区（战场）

1. **区域标识**：
   - 在电脑出牌区上方增加细线横幅：`敌方出牌 // ENEMY PLAY`，`#AAAAAA`，`h * 0.02`。
   - 在玩家出牌区下方增加细线横幅：`我方出牌 // ALLY PLAY`。

2. **出牌区背景**：
   - 在出牌卡牌下方绘制一个淡淡的矩形框 `#0A0A0A`，带 1px `#1A1A1A` 边框，提示此为"战场"区域。

#### 3.7.3 玩家手牌区

1. **标签**：
   - 原 `玩家 N 张` → `我方单位 // PLAYER`
   - 右侧增加手牌数量荧光绿角标。

2. **选中高亮**：
   - 原黄色 overlay `#FFFF64` alpha100 → 改为荧光绿 `#CCFF00` alpha60。
   - 增加外发光效果：在高亮卡牌外缘绘制 2px `#CCFF00` alpha40 的边框（通过放大 1.02 倍的 sprite 绘制实现）。

3. **手牌扇形**：
   - 保留原有扇形旋转和下沉逻辑。
   - 牌背（玩家看不到自己的牌背，但发牌动画涉及）保持原样。

#### 3.7.4 能量条（Energy Bar）

1. **位置**：保持底部中央偏右。
2. **样式改造**：
   - **背景槽**：高度增加至 `h * 0.035`，圆角改为直角，填充 `#111111`，边框 `#333333` 1px。
   - **填充条**：由黄色 `#FFC800` 改为荧光绿 `#CCFF00`。填充方式改为**分段式**：每 1 点能量对应一个独立段，段间有 2px 黑色间隔，类似弹夹/电池格。
   - **文字**：`⚡ {current} / {max}` → `ENERGY: {current}/{max}`，荧光绿，等宽字体，位于能量条右侧。
   - **标签**：能量条左侧增加竖排文字 `PWR` 或横排 `能量 // ENERGY`，`#888888`。

#### 3.7.5 玩家技能槽（左下角）

1. **槽位框架**：
   - 3个槽位作为一个整体，外围增加一个切角矩形面板 `#0A0A0A`，边框 `#333333`。
   - 面板左上角标签：`装备槽 // SLOTS`，`#888888`。

2. **单个槽位**：
   - 形状：切角矩形，比例保持。
   - 空槽：中央绘制小型准星 `#333333`。
   - 有技能：
     - 顶部 6px 色带：可用 `#CCFF00`，能量不足 `#555500`，冷却中 `#333333`。
     - 技能名：白色，字号略微缩小以适应紧凑布局。
     - 图标区：改为线框八角形，内部 `S0{N}` 荧光绿。
     - 底部信息：冷却中显示红色 `CD{N}`，可用状态显示 `费{N}` 荧光绿。

3. **悬停**：槽位整体向右偏移 2px，边框变绿。

#### 3.7.6 出牌 / 不出 按钮

1. **布局微调**：
   - 两按钮整体下移或保持原位，确保不与能量条冲突。
   - 宽度增加至 `w * 0.12`，高度 `h * 0.055`，更扁宽。

2. **不出按钮（SKIP）**：
   - 常态：`#0A0A0A` 填充，`#333333` 2px 边框。
   - 文字：`跳过 // SKIP`，白色。
   - 悬停：边框 `#CCFF00`，文字保持白色。
   - 禁用：填充 `#080808`，边框 `#222222`，文字 `#444444`。

3. **出牌按钮（EXECUTE）**：
   - 常态：`#1A2A0A` 填充，`#CCFF00` 2px 边框。
   - 文字：`执行 // EXECUTE`，`#CCFF00`。
   - 悬停：填充 `#CCFF00`，文字反黑 `#000000`，增加顶部 3px 斜条纹装饰。
   - 禁用：填充 `#0A0A0A`，边框 `#444400`，文字 `#555500`。

4. **悬停缩放**：保留原有 `m_playBtnHoverScale` / `m_passBtnHoverScale` 逻辑，缩放比例范围改为 `1.0f → 1.08f`（更克制）。

#### 3.7.7 状态文字（Status）

1. **位置**：底部中央，`h * 0.96`。
2. **样式**：
   - 增加黑色背景条（高度 `h * 0.04`，宽度自适应 + 40px padding），`#000000` alpha 200，作为文字衬底确保可读性。
   - 文字颜色按状态区分：
     - 玩家回合：`#CCFF00`
     - 敌人回合：`#FF3333`
     - 胜利：`#CCFF00` + 闪烁
     - 失败：`#FF3333` + 闪烁
   - 格式增加前缀：`[STATUS] 你的回合 // YOUR TURN`。

#### 3.7.8 返回与调试按钮

1. **返回按钮**：统一为顶部标签式按钮，`#0A0A0A` 底，`#333333` 边框，`返回 // BACK`。悬停变绿。
2. **调试按钮**：缩小尺寸，颜色改为暗红 `#330000` 底、`#662222` 边框，降低存在感，避免破坏整体风格。

## 五、新增绘制辅助函数建议

// 绘制切角矩形（八边形）

## 八、注意事项

1. **碰撞检测不变**：所有 `hitTest*` 与 `hitTransition*`、`hitReward*` 等函数保持原逻辑，仅当视觉元素位移导致 hit 区域与显示区域严重不符时才同步微调（如按钮向右偏移 2px 时，hit 区域也应同步偏移）。
2. **保持比例布局**：所有坐标仍基于 `winSize.x / winSize.y` 动态计算，不引入固定像素值。
3. **SFML 3.0.2 兼容**：所有绘制仍使用 `sf::RectangleShape`、`sf::Text`、`sf::Sprite`、`sf::RenderTexture`，不引入需要额外库的新特性。
4. **字体回退**：若用户系统无微软雅黑，保留原有的 `simsun.ttc` 回退机制，风格差异可接受。
5. **性能**：网格背景与扫描线若用循环绘制可能增加 draw call，建议预渲染到 `sf::RenderTexture` 并作为静态 Sprite 重复绘制。
